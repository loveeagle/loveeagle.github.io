三位数排序

  三位数排序，两两比较，共有6种可能性。无论代码怎么写，都要覆盖这六种可能结果，才能得到正确结果。

  • a > b > c
  • a > c > b
  • c > a > b
  • b > a > c
  • b > c > a
  • c > b > a

  本题有两个输入数，年份和月份。那么先判断年份还是先判断月份，我们的代码写法有很大差异。

  • 可以先判断年份是否为闰年再判断月份是否为二月。
  • 也可以先判断月份是否为二月，如果是二月再判断年份是否为闰年。

  显然两种判断顺序，先判断月份代码的效率更高一些，如果不是二月，可以直接输出，不需要再判断年份了。如果是二月，再判断年份是否为闰年。

  使用数组的话，空间复杂度为O(n)。再看已经计算过的日期，除了最不高兴的一天，其他时间不会再用到，所以可以不用数组存储下所有的时间安排，只保留最不高兴的一天。
